# 2026-03-26 Daily Notes

## 动植物图鉴 (bio_server.py) - 重要技术细节

### 数据库 Schema
- `openclaw_db.animal` 表：type ENUM('animal','plant')，164条动物，44条植物
- `organism_id` 在 user_sightings 中：动物用 INT（数据库自增ID），植物用 VARCHAR(100)（中文学名）
- 植物的 organism_id 值 = Unicode码点之和的负值（如苏铁=-71568，银杏=-64581），唯一且非INT

### Wikipedia 缩略图接口 (/api/thumb/<name>)
- 服务器无法访问外网（curl/urllib 都报 network unreachable）
- 所以 `/api/thumb` 接口里的 Wikipedia API 调用实际上不会工作
- 前端 JS 的 fetch 也跨域失败
- **实际解决方案**：从服务器内部测试 Wikipedia API 是通的（Python 直接调），但 Flask 进程的 HTTP 请求失败
  - 原因：OpenClaw Gateway 的 NAT/防火墙规则只允许特定端口出站（18547等），不允许 80/443）
- **当前 workaround**：/api/thumb 端点存在，但运行中的服务器无法真正调用 Wikipedia

### 前端缩略图架构
- index.html: 每个卡片有 `<img class="th" data-name="{{it.name_cn}}" data-sci="{{it.scientific_name}}">`
- loadThumbs(): JS 批量查询 /api/thumb/<name>，优先用学名，找不到再用中文名
- showDetail(): 弹窗里也有一张 Wikipedia 大图（用学名查）
- seen.html: 同样结构

### 服务端口
- bio_server.py: port 5000，PID 不固定，keepalive 守护
- 所有页面需要先登录 testuser/test123
